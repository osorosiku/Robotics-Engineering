gpiozero / Motor

Motor ライブラリの使い方

このページは、現在の motor.py に合わせて、Python の gpiozero.Motor の基本的な使い方、配線の考え方、実行例をまとめたものです。

1. 何をするライブラリか

gpiozero.Motor は、2本の GPIO ピンを使って DC モーターの正転・逆転・停止を扱うためのクラスです。 forward と backward の2つのピンを与えることで、Hブリッジなどのモータードライバ経由で制御します。

  • forward() で正転
  • backward() で逆転
  • stop() で停止

2. 現在の配線設定

今の motor.py では、4台のモーターをそれぞれ別の GPIO に割り当てています。

モーター forward backward
motor1 18 23
motor2 27 22
motor3 17 24
motor4 25 8
注意: 実機では GPIO の重複利用や busy 状態があると初期化で失敗します。配線と他プロセスの使用状況を確認してください。

3. 基本的な書き方

1台だけ動かす最小例は次の通りです。

from gpiozero import Motor

motor = Motor(forward=17, backward=18)

motor.forward(1)
# motor.backward(1)
# motor.stop()

4. 現在の motor.py の動き

スクリプトは起動後に4台のモーターをすべて forward(1) で回し続けます。 while True の無限ループなので、止めるには Ctrl + C を使います。 

while True:
    motor1.forward(1)
    motor2.forward(1)
    motor3.forward(1)
    motor4.forward(1)

5. 実行前の準備

  • gpiozero がインストールされていること
  • Raspberry Pi など GPIO を扱える環境であること
  • モータードライバと GPIO の配線が正しいこと
  • 同じ GPIO を他のプログラムが使っていないこと

6. よく使う操作

  • motor.forward(1): 正転。値は速さの目安です。
  • motor.backward(1): 逆転。
  • motor.stop(): 停止。
  • pause(): シグナル待ちでプロセスを生かすときに使います。

7. メモ

このリポジトリの現状では、hoge.py も gpiozero.Motor を使う同系統のサンプルです。 まずは1台だけで動作確認し、その後に4台構成へ広げると切り分けしやすくなります。
